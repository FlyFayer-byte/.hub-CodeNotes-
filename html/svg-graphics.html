HTML Графіка SVG

SVG (масштабована векторна графіка)

SVG визначає векторну графіку в XML, яку можна безпосередньо вставляти в сторінки HTML.

Графіка SVG є масштабованою та не втрачає якості, якщо її масштабувати або змінювати:

SVG підтримується всіма основними браузерами.

Що таке SVG?

  • SVG розшифровується як Scalable Vector Graphics - Масштабована векторна графіка
  • SVG використовується для визначення векторної графіки для Інтернету
  • SVG визначає графіку у форматі XML
  • Кожен елемент і атрибут у файлах SVG можна анімувати
  • SVG інтегрується з іншими стандартами, такими як CSS, DOM, XSL і JavaScript

HTML елемент <svg>

Елемент HTML <svg>> є контейнером для графіки SVG.

SVG має кілька методів для малювання контурів, прямокутників, кіл, багатокутників, тексту та багато іншого.

Коло SVG

Приклад: HTML 
 
<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
</svg>

Відмінності між SVG і Canvas

SVG — це мова для опису 2D-графіки в XML, тоді як Canvas малює 2D-графіку на льоту (за допомогою JavaScript).

SVG базується на XML, що означає, що кожен елемент доступний у SVG DOM. Ви можете приєднати обробники подій JavaScript до графіки SVG.

У SVG кожна намальована фігура запам’ятовується як об’єкт. Якщо атрибути об’єкта SVG змінено, браузер може автоматично повторно відобразити форму.

Полотно відтворюється піксель за пікселем. У Canvas, як тільки графіка намальована, вона забувається браузером. Якщо його положення потрібно змінити, необхідно перемалювати всю сцену, включаючи будь-які об’єкти, які могли бути покриті графікою.

Порівняння SVG і Canvas

У таблиці нижче показано деякі важливі відмінності між Canvas і SVG:

Canvas SVG
  • Залежить від роздільної здатності
  • Немає підтримки обробників подій
  • Низькі можливості відображення тексту
  • Ви можете зберегти отримане зображення як .png або .jpg
  • Добре підходить для графічних інтенсивних ігор
  • Не залежить від роздільної здатності
  • Підтримка обробників подій
  • Найкраще підходить для програм з великими областями рендерингу (Карти Google)
  • Повільний рендеринг, якщо комплекс (все, що використовує DOM) буде дуже повільним
  • Не підходить для ігрових програм
